# Основные команды Docker Compose

Этот файл — краткая шпаргалка по наиболее часто используемым командам `docker-compose` для управления проектом.

---

### 1. Запуск сервисов

Эта команда собирает, создает, запускает и связывает все сервисы, описанные в вашем файле `docker-compose.yml`.

```bash
# Запустить все сервисы в фоновом режиме
docker-compose up -d
```

*   **Полезные флаги:**
    *   `--build`: Принудительно пересобрать образы перед запуском. Используйте, если вы внесли изменения в `Dockerfile` или код приложения.
    *   `-d` (`--detach`): Запустить контейнеры в фоновом режиме. Без этого флага вы увидите логи всех сервисов прямо в терминале.

---

### 2. Остановка сервисов

Эта команда останавливает и удаляет контейнеры, сети и другие ресурсы, созданные командой `up`.

```bash
# Остановить и удалить все запущенные сервисы
docker-compose down
```

*   **Полезные флаги:**
    *   `-v` (`--volumes`): Удалить также и тома (volumes). **Осторожно:** это приведет к удалению данных, например, вашей базы данных.

---

### 3. Просмотр логов

Показывает вывод (логи) от запущенных сервисов. Незаменимая команда для отладки.

```bash
# Показать логи всех сервисов и следить за ними в реальном времени
docker-compose logs -f
```

*   **Полезные флаги:**
    *   `-f` (`--follow`): Следить за логами в реальном времени.
    *   `[имя_сервиса]`: Можно указать имя конкретного сервиса, чтобы видеть только его логи (например, `docker-compose logs -f dmkod-integration-app`).

---

### 4. Просмотр статуса контейнеров

Показывает список контейнеров, запущенных для вашего проекта, и их текущий статус.

```bash
docker-compose ps
```

---

### 5. Выполнение команды внутри контейнера

Позволяет выполнить разовую команду внутри работающего контейнера. Чаще всего используется для доступа к командной строке.

```bash
# Открыть интерактивную оболочку (shell) внутри сервиса с именем 'app'
docker-compose exec app /bin/sh
```
